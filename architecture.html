핵심 설계 결정

🌐도메인 분리 가능

domains/<name>/keywords.yaml + sources.yaml 만 갈아끼우면 다른 토픽 (헬스케어/핀테크 등) 으로 재사용. 코드 수정 X.

🔒schema 격리

모든 테이블을 ai_news schema 에 두어 같은 Supabase 프로젝트의 다른 서비스와 충돌 없음.

🎯3위일체 검색

키워드 + 매체 화이트리스트 (site:) + 최신순 (freshness=pd) 한 쿼리에 묶음 → 키워드당 1 호출.

🛡️2단 방어 본문 추출

URL 패턴 휴리스틱 (카테고리/홈페이지 차단) + 본문 길이 ≥ 500자 검증 → 메일에 매체 인덱스 페이지 안 섞임.

🇰🇷통합 정리, 개별 번역 X

Gemini 가 10~20개 영문 본문을 받아 그날의 AI 트렌드를 한국어 마크다운 단일 디지스트로 정리. 기사별 번역 X.

🔄완전 무료 운영

Brave Search (월 2,000 cap), Supabase (500MB), Gemini Flash, Gmail SMTP, GitHub Actions cron — 전부 free tier 안.

5단계 파이프라인 (자세히)

각 단계 카드를 클릭하면 입력/출력/실패 처리 + 코드 위치가 펼쳐집니다.

1
검색 — Brave Search 3위일체 키워드 × 매체 × 최신순 → URL 리스트
›
입력

keywords (5개) × source_domains (10개)

호출

키워드별 1회. 호출당 매체 10개를 site:(d1 OR d2 ... OR d10) 로 한 쿼리에 묶음.

GET https://api.search.brave.com/res/v1/web/search
?q="artificial intelligence" (site:techcrunch.com OR site:theverge.com ...)
&count=10
&freshness=pd        # past day
Headers: X-Subscription-Token: $BRAVE_SEARCH_API_KEY
필터 (2단)

① meta_url.hostname 으로 화이트리스트 재검증
② URL 패턴 휴리스틱 — 카테고리/홈페이지 (/blog/, /category/, segment < 2 등) 제외

출력

SearchResult(url, title, snippet, source_domain, keyword) list

Rate limit 보호

Brave Free = 1 query/sec. pipeline.run() 이 키워드 사이 1.2초 sleep.

실패 처리

한 키워드 실패 (네트워크/429) 시 그 키워드만 skip, 다른 키워드는 계속.

본문 추출 — trafilatura + 브라우저 UA URL → 영문 본문 텍스트 + 메타데이터
›
입력

1단계의 URL list (dedup 후 새 URL 만)

Dedup

store.existing_urls(urls) 로 articles 테이블 IN 조회 → 이미 있는 URL 제외 → 새 URL 만 fetch.

Fetch

requests.get(url, headers=BROWSER_HEADERS) — User-Agent: Chrome 120 + Accept-Language: en-US.

Headers:
  User-Agent: Mozilla/5.0 (Macintosh; ... Chrome/120.0.0.0 Safari/537.36)
  Accept: text/html,application/xhtml+xml,...
  Accept-Language: en-US,en;q=0.9
Extract

trafilatura.extract(html) → 본문 텍스트
trafilatura.extract_metadata(html) → title, date

품질 검증

본문 길이 ≥ MIN_BODY_LEN (500자) 이어야 통과. 미달이면 ExtractionError (카테고리 페이지 2단 방어).

출력

ExtractedArticle(url, title, body_text, raw_html_excerpt, published_at, source_domain)

실패 처리

403 / paywall / JS 동적 렌더 매체는 자동 skip + 로그.

›
입력

최대 20건 (max_articles_for_summary) 의 SummaryInput(title, source_domain, url, body_text, published_at)

현재 모델

gemini-3.5-flash (Google AI Studio 무료 tier). 변경은 .env 의 GEMINI_MODEL.

프롬프트 (5 규칙)
1. 개별 기사 번역이 아닌, 트렌드 단위로 통합 정리하라.
2. 비슷한 주제는 한 단락으로 묶고, 출처 매체를 함께 명시하라.
3. 출근 전 5분 안에 읽을 수 있는 분량으로 작성하라.
4. 각 트렌드 끝에 출처 링크를 [매체명](URL) 형식으로 첨부하라.
5. 출력은 한국어 마크다운만. 영어 원문 인용 최소화.
컨텍스트 보호

기사당 본문 4000자 cap. 20개 × 4000 = 80,000자 ≈ 20,000 토큰 (Gemini 1M context 의 2%).

데이터 모델 (ai_news schema)

Supabase Postgres. 모든 테이블에 RLS 활성화 — service_role 키만 우회 가능.

ai_news.articles

  • id bigserial
  • url text unique
  • title text
  • source_domain text
  • source_name text
  • published_at timestamptz
  • body_text text
  • raw_html_excerpt text
  • keyword text
  • run_id uuid
  • fetched_at timestamptz

ai_news.subscribers

  • id bigserial
  • email text unique
  • active boolean
  • created_at timestamptz
  • updated_at timestamptz

ai_news.runs

  • run_id uuid
  • started_at timestamptz
  • finished_at timestamptz
  • article_count int
  • status text
  • error text
  • digest_text text

ai_news.scrape_enabled

  • id int (always 1)
  • enabled boolean
  • updated_at timestamptz
🔒 RLS: 4 테이블 모두 enable row level security. 별도 policy 없음 → service_role 키 (RLS 우회 권한) 만 통과. anon/authenticated 키로는 어떤 read/write 도 불가능.
🏷️ schema 격리: public 이 아닌 ai_news schema 에 둠. 같은 Supabase 프로젝트에 다른 서비스를 올려도 충돌 0. 반드시 Dashboard → Settings → API → "Exposed schemas" 에 ai_news 추가 필요.

운영

스케줄

cron: "40 23 * * *"   # UTC = 23:40 → KST 08:40 (다음날)
GitHub Actions runner: ubuntu-latest, timeout 15분

GitHub Actions cron 은 부하 높을 때 ~15분 지연 가능 — CLAUDE.md §4 의 ±30분 허용 범위 안.

매일 실행 흐름

  1. cron trigger → uv sync --frozen
  2. python -m ai_news_scraping.cli run
  3. scrape_enabled 토글 확인 (OFF 면 skip + exit 0)
  4. subscribers 명단 조회 (active=true)
  5. 비어 있으면 skip + exit 0
  6. pipeline.run() — 5 단계
  7. SMTP 발송 결과 로그

수동 트리거

GitHub Actions → Daily AI News Digest → Run workflow
- inputs.dry_run = true / false
- 즉시 실행 (cron 대기 X)

일시 중단

로컬 admin 페이지 (http://127.0.0.1:6661) 의 "토글" 버튼 → OFF. 다음 cron 부터 자동 skip. 재개는 토글 ON.
